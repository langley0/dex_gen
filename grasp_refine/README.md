# grasp_refine

`grasp_refine` is the post-processing stage for grasp poses generated by `grasp_gen`.
The previous refine logic was removed and replaced with a DexGrasp Anything (DGA)-style
physics-guided sampling flow.

## What The Current Refine Does

The current pipeline is split across these files:

- `grasp_refine/refine.py`
- `grasp_refine/batch.py`
- `scripts/_grasp_refine_bridge.py`
- `scripts/run_grasp_refine.py`
- `scripts/run_grasp_refine_batch.py`
- `scripts/demo_grasp_refine_dga.py`

At a high level:

1. Start from a source hand pose from `grasp_gen`.
2. Build physics-inspired objective terms.
3. Evaluate the objective gradient with JAX.
4. Mix a random sampling direction and a physics gradient direction.
5. Project the sampled hand pose back to a valid hand parameterization.
6. Keep the lowest surrogate-energy pose as the current `best`.

The guided update is DGA-style:

- `d_sample`: stochastic exploration direction
- `d_star`: normalized negative physics gradient direction
- `d_mix = d_sample + guidance_scale * (d_star - d_sample)`

The actual step size is additionally modulated by per-parameter `step_scales`
and `grad_scale`.

## Current Physics Terms

The current refine objective uses three terms:

### 1. Surface Pull

This term is the current "contact maintenance" force.

- It is **not** inverse-distance.
- It is **not** deepest-point-only.
- It activates only for selected hand contact points that are already within
  `surface_pull_threshold` of the object surface.
- The active points contribute their unsigned nearest-surface distance, averaged
  over the active set.

Important consequence:

- If `surface_pull_weight` is too large, the hand can keep trying to stay glued
  to the surface even while overlap should be released first.

### 2. External Repulsion

This term is the current hand-object overlap penalty.

- A dense hand cloud is queried against the object mesh.
- A normal-based signed test is used to detect points that are effectively
  inside the object.
- The current implementation uses the **maximum** penetration-like value across
  the hand cloud, so it behaves like a worst-point penalty.

Important consequence:

- This is closer to the DGA paper's max-based external penetration term than
  surface pull is.
- However, a single worst-point penalty can still leave many shallow overlaps
  under-corrected.

### 3. Self Repulsion

- A reduced set of representative hand points is used.
- Pairwise distances below `self_repulsion_threshold` are penalized.

## How To Read `actual_*` Metrics

The `actual_*` fields are not surrogate loss terms.
They come from the MuJoCo collision scene in `grasp_sampling/scene.py`.

- `actual_contact_count`: total hand-object contacts
- `actual_penetration_count`: contacts with positive penetration depth
- `actual_depth_sum`: sum of positive penetration depth
- `actual_max_depth`: maximum positive penetration depth

If `best actual penetration_count` or `best actual depth_sum` goes up after
refine, then overlap got worse in the actual physics scene, even if the
surrogate objective improved.

## Important Difference Between Paper Intent And Current Behavior

The DGA paper uses:

- Surface Pulling Force (SPF): near-surface pulling over a set of points close
  to the object surface
- External-penetration Repulsion Force (ERF): a max-based signed-distance
  penalty
- Self-Penetration Repulsion Force (SRF): thresholded pairwise hand-point
  repulsion

The current implementation follows that structure at a high level, but there are
still practical gaps:

- `best_hand_pose` is selected by surrogate total energy, not by actual MuJoCo
  overlap quality.
- `surface_pull` can be too aggressive when the real goal is "push only until
  the overlap reaches the surface."
- `contact_target_local` is carried through the API but is not yet used as an
  explicit anchor term in the objective.
- `external_threshold` exists in config/CLI but is not yet fully wired into the
  single-sample energy computation as a true deadband.

## Practical Tuning Guidance

If overlap is getting worse after refine, start here:

- lower `surface_pull_weight`
- lower `surface_pull_threshold`
- raise `external_repulsion_weight`
- lower `grad_scale`
- lower `guidance_scale`
- reduce sampling noise late in the schedule

Example:

```bash
.venv/bin/python scripts/demo_grasp_refine_dga.py \
  --regenerate \
  --surface-pull-weight 0.05 \
  --external-repulsion-weight 2.0 \
  --surface-pull-threshold 0.002 \
  --grad-scale 0.03 \
  --guidance-scale 0.5 \
  --noise-scale-start 1e-3 \
  --noise-scale-end 1e-4
```

## Recommended Next Direction

If the real goal is:

> "Only push overlapped geometry out until it reaches the surface."

then the most promising next changes are:

1. Make `best` selection actual-overlap-aware.
2. Gate or disable surface pull while actual overlap still exists.
3. Add a surface-band objective that stops applying force once the point reaches
   a thin shell around the object surface.
4. Consider a two-stage refine:
   overlap release first, gentle contact recovery second.
